# ProxyTor Gateway v0.1.0

Initial public release of ProxyTor Gateway.

## Highlights

- Tor SOCKS5 gateway support.
- Privoxy HTTP proxy forwarding through Tor.
- FastAPI dashboard/API.
- Admin and viewer token model.
- Telegram bot integration.
- Dynamic token rotation support.
- SQLite persistence.
- Audit and event logging.
- CSV/JSON audit export by date range.
- Recent client tracking.
- Abuse detection based on connection thresholds.
- Ban/unban workflows through dashboard and Telegram.
- systemd service units.
- Example configuration files for Tor, Privoxy, ProxyTor and Telegram.
- GitHub Actions validation workflow.

## Security defaults

- Production secrets are excluded from the repository.
- Example configuration uses placeholders instead of real environment values.
- Tor exit IP change Telegram alerts are disabled by default to reduce alert noise.
- Proxy ports should remain restricted to trusted networks.

## Validation

The project includes GitHub Actions checks for:

- Python syntax validation.
- Shell script validation with ShellCheck.
- Basic public secret/private value scan.

## Upgrade notes

This is the first public release. Existing private/lab deployments should compare local configuration files before replacing production files.

## Known limitations

- Installer is intended for Debian 12 Bookworm.
- Reverse proxy TCP streams may hide the original client IP.
- Device identification is best-effort and depends on available host/MAC/vendor data.
