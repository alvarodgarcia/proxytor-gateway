# ProxyTor Gateway v0.1.1

Maintenance release focused on installer safety, public documentation quality and repository validation.

## Release type

Maintenance / documentation / operational hardening.

## Highlights

- Safer and more idempotent installer.
- Dashboard screenshot gallery documented.
- GitHub Actions validation workflow documented as completed.
- README roadmap updated.
- Public release notes added.

## Changes

### Installer

- Added safer re-run behaviour to `scripts/install.sh`.
- Existing admin and viewer tokens are preserved.
- Existing `/etc/proxytor-api/config.json` is preserved.
- Existing `/etc/default/proxytor-telegram` is preserved.
- Existing `/etc/tor/torrc` is preserved by default.
- Existing `/etc/privoxy/config` is preserved by default.
- ProxyTor Tor and Privoxy examples are copied separately:
  - `/etc/tor/torrc.proxytor.example`
  - `/etc/privoxy/config.proxytor.example`
- Added centralized install backups under:
  - `/root/proxytor-install-backups/`
- Added installer options:
  - `--dry-run`
  - `--skip-packages`
  - `--force-config`
  - `--help`
- Added Tor configuration validation before service restart.
- Added Privoxy configuration validation when supported.
- Reuses the existing Python virtual environment when present.

### Documentation

- Added dashboard preview at the top of the README.
- Added link to the full dashboard screenshot gallery.
- Documented safer installer behaviour.
- Documented installer options.
- Added installer commands to Useful Commands.
- Updated roadmap status:
  - More robust installer idempotency: completed.
  - Dashboard screenshots: completed.
  - GitHub Actions linting: completed.

### CI / validation

- Repository validation includes:
  - Python compile checks.
  - ShellCheck for shell scripts.
  - Basic scan for forbidden public values.
- GitHub Actions uses current checkout action.

## Upgrade notes

From an existing v0.1.0 installation:

~~~bash
git pull
sudo bash scripts/install.sh --skip-packages
~~~

The installer preserves existing runtime configuration by default.

Use this only if you intentionally want to replace Tor and Privoxy configs with the repository examples:

~~~bash
sudo bash scripts/install.sh --force-config
~~~

Recommended review after upgrade:

~~~bash
systemctl status tor@default --no-pager
systemctl status privoxy --no-pager
systemctl status proxytor-api --no-pager
~~~

## Security notes

This release does not change the exposure model.

Do not expose these ports directly to the Internet:

- `9050/tcp`
- `8118/tcp`
- `8088/tcp`

Use VPN, private access or a properly protected reverse proxy for dashboard access.

## Known notes

- Client usage registry and advanced device fingerprinting are not included in v0.1.1.
- Those features are being evaluated separately in lab and should be considered candidates for a later release.
